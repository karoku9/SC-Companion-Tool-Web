# v0.22 Expanded Universe Data

## Release goal

Make ordinary hauling destinations resolve through one maintainable registry before Game.log intake is introduced. A parsed name must map to a stable operational destination, hierarchy, in-game navigation target, source record and Starmap anchor without inventing precision or facility services.

## Delivered coverage

- 80 normalized location records.
- 34 operational destinations: 30 in Stanton, 3 in Pyro and 1 in Nyx.
- Four Stanton landing-zone arrival points and four orbital stations.
- Grim HEX.
- Eighteen Stanton Lagrange stations with punctuation-tolerant aliases.
- Pyro, Magnus and Terra gateways in Stanton.
- Existing Checkmate, Orbituary, Ruin Station and Levski coverage retained.

## Data boundaries

- Official RSI references remain marked as official.
- The current Stanton destination snapshot is marked as reviewed community game data.
- Every record carries a game version and review date.
- Physical body mappings, schematic offsets, Lagrange anchors and gateway anchors have distinct geometry statuses.
- Registry presence never implies live traffic, security, market or service telemetry.
- New community-backed locations receive source profiles but no invented facility services.
- Selected-system context in the Network layer states the registry version and review date without presenting schematic topology as live data.

## Validation contract

- Unique source and location IDs.
- Every parent, source and anchor body must exist.
- Every operational destination must expose a navigation target and map anchor.
- Exact and common punctuation variants must resolve predictably.
- Every anchor must produce finite map and distance coordinates.
- Mission parsing must reuse the same resolver and avoid custom locations for supported aliases.
- Chromium must generate an expanded Stanton itinerary plus desktop, tablet and mobile visual artifacts.
- Node and Chromium workflows must remain green.
- Desktop, tablet and mobile screenshots must be inspected before merge.

The release remains a candidate until the complete workflow and generated screenshots satisfy this contract.
